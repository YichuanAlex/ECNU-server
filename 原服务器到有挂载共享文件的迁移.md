# 原服务器到有挂载共享文件的迁移

目前的解决办法是：
1. 创建一个新的容器挂载到共享文件夹上
2. 把原服务器上的内容通过scp命令进行传输

```
scp -r -P [目标服务器的端口号] [所在服务器的目录或文件] root@49.52.20.55:[目标服务器存储地址]

e.g.
scp -r -P 100 temp_dir root@49.52.20.55:/root/

把当前服务器目录的temp_dir 传输到 端口号为100的容器的/root/文件夹下 
```

3. 迁移完毕之后，把原容器进行删除。


## 新容器配置：

参考[docker容器配置方法](https://github.com/PLANING-lab/server/blob/main/docker%E5%AE%B9%E5%99%A8%E9%85%8D%E7%BD%AE%E6%96%B9%E6%B3%95.md)

1. 安装Miniconda: `sh Miniconda3-latest-Linux-x86_64.sh`，然后执行`vim ~/.bashrc`，文件下添加`export PATH=/root/miniconda3/bin:$PATH`，然后执行`source ~/.bashrc`，之后执行`conda --version`，如果输出了conda的版本号，即为安装成功。然后执行`conda init`，之后再次执行`source ~/.bashrc`来更新系统环境，完成后，即可使用`conda create -n env_name python=x.xx`来创建conda环境

2. 安装cuda: `sh cuda_版本号_linux.run`, 文件夹中准备了cuda12.4,12.2,12.1三个版本，可以按需安装。在执行命令后需要输入`accept`，然后在选项中去掉第一个driver驱动后安装即可。在安装后需要将其加入环境变量，先执行：`vim ~/.bashrc`，在文件下添加
```
export PATH=/usr/local/cuda-12.x/bin${PATH:+:${PATH}}
export LD_LIBRARY_PATH=/usr/local/cuda-12.x/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}}
export CUDA_HOME=/usr/local/cuda-12.x
```
将上面代码中的`12.x`替换为你实际安装的版本号。保存后，更新系统环境，执行`source ~/.bashrc`。这时输入`nvcc -V`，如果输出了关于CUDA的版本信息，即为安装成功。

3. 安装pytorch: pytorch版本需要与cuda相一致，请访问https://pytorch.org/get-started/previous-versions/ 获取你所需要的版本以及安装方式。在安装完成后，可以通过执行
```
python
import torch
torch.__version__
torch.cuda.is_available()
```
来检查是否安装了可以使用GPU的torch版本。
